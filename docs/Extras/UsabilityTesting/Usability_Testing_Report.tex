\section{Usability Goals}
Throughout the design process we aimed to acheive the following usability goals which are tied to
our non functional requierement NFR.S.2: Provide an intuitive interface accessible to users
with minimal training. Here are the goals:
\begin{itemize}
    \item Goal 1: User interface is easy to use and navigate for users of all experience levels.
    \item Goal 2: Board cleary and intuitvley represents actual catan board, allowing easy playing and learning.
    \item Goal 3: Teach users catan through experiential learning.
\end{itemize}

\section{Elicitation Plan \& Instruments}
\subsection{Elicitation Plan: Observational Think-Aloud Study}
To gather qualitative data on user interaction with the Catan Arena interface, we implemented an observational think-aloud study. This method allows us to identify specific friction points in the UI and verify if the catan arena actually aid the "Experiential Learning" cycle.
\medskip   

\textbf{Tesing Goals:}
\begin{itemize}
    \item Identify usability challenges in the user interface.
    \item Measure the effectiveness of experiential learning.
    \item Assess the overall user experience and satisfaction with the interface, especially in terms of learning and engagement.
\end{itemize}

\textbf{Observed User Groups:}
\begin{itemize}
    \item \textbf{Beginners:} Users with little to no experience playing Catan, who are new to the game and may be unfamiliar with basic mechanics.
    \item \textbf{Novice Players:} Users who have some experience with Catan, understand basic rules, but may struggle with strategy and resource management.
    \item \textbf{Competitive Players:} Experienced users who have a strong understanding of Catan mechanics and strategy.
\end{itemize}

\textbf{Observation Setting:}
\begin{itemize}
    \item \textbf{Remote:} Conducted via screen-share, where users are encouraged to "think aloud" while navigating the web-based interface.
\end{itemize}

\textbf{Observation Tasks:}
\begin{enumerate}
    \item \textbf{Game Setup:} Start a new game human vs. bot game (Final Boss) with default settings and applicable flesch score.
    \item \textbf{Game} Play a full game against Final boss.
    \item \textbf{Replay}: When game finishes, use the replay feature to review the game.
    \item \textbf{Move Cycling:} Go through different moves in the game.
    \item \textbf{Bot Vs Bot}: Start a new game where 2 different bots play against each other.
    \item \textbf{Analyze} During the bot vs bot game, use analyze feature to see the most likely to win bot.
    \item \textbf{Bot Vs Bot Replay}: After the bot vs bot game finishes, use the replay feature to review the game.
    \item \textbf{Bot Vs Bot Move Cycling}: Go through different moves in the bot vs bot game.
\end{enumerate}

\textbf{Instruments Used:}
\begin{itemize}
    \item \textbf{Demographic Questionaire:} A pre-observation questionnaire to collect demographic information and Catan Experience.
    \subitem \textbf{Demographic Questions:}
    \begin{itemize}
        \item \textbf{Age:} "What is your age?"
        \item \textbf{Gender:} "What is your gender (Male,Female,Other)?"
        \item \textbf{Catan Experience:} "How would you say your experience with Catan is? (Beginner, Novice, Competitive)"
    \end{itemize}
    \item \textbf{Researcher Notes Template:} Notes for the interviewer to record observations, user comments, and emotional responses during the think-aloud sessions.
    \item \textbf{Post-Observation Survey:} A post observation survery with a open ended question and 1-5 rating to quantify Clarity, Game Flow, and Intuition.
    \subitem \textbf{Survery Questions:}
    \begin{itemize}
        \item \textbf{Overall Experience:} "Overall, how was your experince with the interface? Please share any general recomendations or feedback you have about the usability and design of the interface."
        \item \textbf{Clarity:} "On a scale of 1 to 5, how clear was the interface with what is happening in the game, allowing you to easily understand the game state and the actions being taken?"
        \item \textbf{Game Flow:} "On a scale of 1 to 5, how well did the interface support the flow of the game, allowing you to easily follow and take actions without feeling lost or confused?"
        \item \textbf{Intuition:} "On a scale of 1 to 5, how intuitive was the interface in matching the mechanics of the board game, making it easy for you to learn and play without needing extensive instructions?"
    \end{itemize}
\end{itemize}

\section{Elicitation Process}
This section details the elicitation process of 7 participants across the three user groups (Beginners, Novice Players, Competitive Players) using the observational think-aloud method. Participants were observed while interacting with the Catan Arena interface, following the observation tasks set out in the elicitation plan. After the observation, participants completed a post-observation survey to help quantify their experince and also get recommendations for improvement. The following subsections provide a detailed breakdown of each participant's experience and feedback. If needed, a further interview can be conducted to re evaluate participants after the UI changes to see if their experience  has improved.

\section{Design Changes Based on Elicitation Results}

\subsection{Features Added/Changed}
\begin{itemize}
    \item \textbf{Board Intuition:} Users didnt like the patterns for the tiles, and found it hard to match the UI to the real board game, which was a barrier to learning and immersion.
    \subitem Changed patterns to be more similar to the board game, and changed colours to be more similar to the board game.
    \item \textbf{Action Log:} Users found the action log to be too small and cluttered, making it difficult to follow the game flow and understand what actions were being taken.
    \subitem Increased the size of the action log and improved the layout to make it easier to read and follow.
    \item \textbf{Fixed Drawers:} One user was confused by the blank right side of the screen during the analyze feature, and another user tried to slide the drawer closed after using it, and multiple users tried to zoom in on the board and were disappointed that it was not possible. 
    \subitem Made the right drawer collapsible, allowing users to hide it when they want a full view of the board. Also made the board zoomable, so users can focus on specific areas of the board without being hindered by the fixed layout.
    \item \textbf{Bot Identification:} Users found it hard to distinguish between the bots during the fast-paced bot vs bot game due to the lack of distinct headers or titles.
    \subitem Added clear headers making it easier for users to track which bot is which and follow the game flow.
    \item  \textbf{Dice Roll Visualization:} Users expressed frustration with having to look at the action log to see dice rolls, stating that dice should be a primary visual element on the board.
    \subitem Added a visual representation of dice rolls directly on the board, making it easier for users to quickly understand the game state without having to refer to the action log.
    \item \textbf{Bot Move Clarity:} Users were confused by why bots made certain moves, indicating a need for better explanations or visual cues to clarify bot decision-making.
    \subitem Added a large feature that allows users to click on any move and see an explanation of why the bot made that move, including the bot's current strategy and how it evaluates the game state. This helps experintal learning.
    \item \textbf{Replay}: We added a button for replay instead of just having it cycle into it after a game finishes. 
    \subitem This was a suggestion from a user who wanted to be able to replay the game at any point, instead of having to watch the whole game and then replay it. This allows users to go back and review specific moments in the game that they found interesting or confusing, enhancing the learning experience. Also if users go back they can still enter the game they were just in.
\end{itemize}

\section{Conclusion}

After implementing the UI changes based on the observational think-aloud study, we observed significant improvements across all usability components. The average scores for Clarity, Game Flow, and Catan-to-Bot Intuition increased from 3.6, 4.1, and 2.8 to 3.9, 4.9, and 4.4 respectively. This indicates that the UI changes successfully addressed the major friction points identified in the initial study, leading to a more intuitive and engaging user experience that better supports experiential learning. Also the new LLM feature seemed to be very useful for users to understand the bot's decision making process, which is a key part of experiential learning. Overall, the iterative design process and user feedback were crucial in enhancing the usability and educational value of the Catan Arena interface.
